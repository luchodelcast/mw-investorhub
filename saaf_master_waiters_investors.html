02 · La solución tecnológica

SAAF no reemplaza a las personas. Las multiplica.

Superlikers Autonomous Agents Framework es una capa de agentes autónomos que se sienta encima de la plataforma existente. No modifica el motor de gamificación, ni las apps de meseros, ni los endpoints que ya operan en producción. Lo complementa con inteligencia proactiva.

Principio de diseño irrenunciable: SAAF nunca publica una oportunidad sin supervisión humana. Siempre hay al menos un humano en el loop antes de que un reto sea visible para los meseros. La autonomía tiene límites deliberados — el administrador de cada sucursal tiene veto final, siempre.

Los tres componentes centrales del sistema:

Componente 1
Agente Proponente
Analiza 12+ meses de historial del POS, la temporada, el portafolio de la marca y los objetivos del mes. Genera una parrilla completa — mínimo 10 retos, 4 desafíos y 2 misiones — sin que nadie se lo pida. Propone 2–3 alternativas por oportunidad para que el humano elija.
Componente 2
Motor de Aprobación
Gestiona el flujo de revisión humana configurable por cliente. Para retos del restaurante, el admin aprueba explícitamente. Para retos de marca, Coca-Cola aprueba primero y el admin tiene 24h para vetar. El admin siempre tiene la palabra final.
Componente 3
Agente Publicador
Toma la propuesta aprobada, construye el payload completo y lo publica vía los endpoints de la plataforma Superlikers que ya existen en producción. Monitorea continuamente y despublica automáticamente si detecta anomalías técnicas.
Motor de aprendizaje
El agente mejora con cada ciclo
Los resultados reales (tasa de cumplimiento, uplift real vs proyectado, tasa de aceptación) retroalimentan al agente antes de la siguiente parrilla mensual. El criterio del Engagement Manager también se incorpora como señal editorial. Cuantos más restaurantes, mejor el agente.

04 · Jerarquía de agentes

Un agente por nivel. Todos proponen. El admin siempre decide.

La arquitectura multi-agente de SAAF espeja la jerarquía real del negocio. No hay un solo agente genérico — hay instancias específicas que entienden el contexto de cada nivel.

06 · La ventaja competitiva

El moat que no se puede copiar fácilmente.

SAAF no es solo un sistema de automatización. Es un volante que gira más rápido con cada restaurante que se incorpora. La data de 1M de restaurantes hace al agente exponencialmente más preciso que el de cualquier competidor que empiece desde cero.

Ventaja 1 · Data flywheel
El agente mejora con escala
Cada reto publicado genera señales de aprendizaje. Con 1M de restaurantes, SAAF tiene acceso a patrones de comportamiento de 50M de meseros globalmente. Ningún competidor puede replicar esa base de datos sin años de operación.
Ventaja 2 · Integración profunda
El POS ya está conectado
La conexión al POS de HOT en tiempo real ya opera en producción. Replicar esa integración en 50+ sistemas de POS globales requiere años de trabajo técnico y acuerdos comerciales que Master Waiters ya está construyendo.
Ventaja 3 · Autonomía con control
Diseño que respeta la autonomía del cliente
El veto del administrador de sucursal no es una concesión — es el diferenciador que HOT Restaurantes exige para escalar. Los competidores que ofrezcan automatización sin control perderán a los operadores más grandes.
Ventaja 4 · Arquitectura genérica
Un sistema, infinitos ecosistemas
SAAF no está hardcodeado para restaurantes. La misma arquitectura sirve a OXXO Engage (28K tiendas), clientes corporativos y cualquier ecosistema gamificado. Cada cliente nuevo fortalece el sistema para todos los demás.

07 · Stack tecnológico y costos

Costos de inferencia manejables incluso a escala completa.

El stack tecnológico fue seleccionado para operar de forma asíncrona desde el principio — la arquitectura que funciona a 23 sucursales funciona a 5,000 sin cambios estructurales.

LLM principal
Claude Sonnet (Anthropic)
200K tokens de contexto — suficiente para pasar 12+ meses de POS + instrucciones en una sola llamada. Español excelente. Razonamiento estructurado consistente para optimización de presupuesto.
Infraestructura
AWS · Python / FastAPI
Step Functions para el flujo de aprobación configurable por cliente. Lambda + Batch para generación en paralelo a escala. SQS para desacoplar generación de procesamiento. RDS PostgreSQL para estado y configuración.
